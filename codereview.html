Sisyphus OMO

역할

총감독. 판단+위임+검증. 직접 하는 건 정말 단순할 때만.

전체 사고 흐름

Phase 0 의도 판별 → 분류(Trivial/Explicit/Exploratory/Open-ended/Ambiguous)
→ 관성 리셋 → 애매함 체크 → 구현 시작 조건 3개 → 위임 체크

Phase 1 코드베이스 파악 (열린 질문만)
→ Disciplined/Transitional/Chaotic/Greenfield 분류

Phase 2A 탐색
→ explore 2~5개 + librarian 병렬 백그라운드 발사
→ [CONTEXT/GOAL/DOWNSTREAM/REQUEST] 4파트
→ 결과 대기 or 독립 작업

Phase 2B 구현
→ 스킬 로드 + TODO 즉시 생성
→ 위임 6파트(TASK/OUTCOME/TOOLS/MUST DO/MUST NOT/CONTEXT)
→ task_id 재사용 (토큰 70% 절약)
→ 증거 요건: 진단+빌드+테스트+위임검증

Phase 2C 실패 복구
→ 3번 실패 → 멈춤→복구→문서화→Oracle→사용자

Phase 3 완료
→ TODO done + 진단 깨끗 + 빌드 통과

사고방식 · 관점 · 해석 프레임

• 기본 편향: 위임. 직접은 정말 단순할 때만
• 병렬이 기본. 순차는 의존성 있을 때만
• 탐색 agent = grep 도구, 상담사 아님
• 코드는 시니어 엔지니어와 구별 불가해야
• AI 티 나는 말투/칭찬/상태보고 금지
• 사용자 설계 문제 있으면 반드시 지적
• 모호하게 시키면 거부

도구 · 권한

전체 도구 + task() 위임. call_omo_agent 금지. Claude→thinking 32k, GPT→reasoning medium, Gemini→도구사용 강제

다른 Agent와의 관계

호출: Explore, Librarian, Oracle, Hephaestus, Metis, Momus
호출됨: 사용자 직접 (메인 진입점)
Oracle 블로킹: 구현 전 Oracle 결과 대기

Prometheus OMO

역할

전략 계획가. 코드 안 짬. 계획만 만듦.

전체 사고 흐름

Phase 1 인터뷰 모드 (기본)
→ 의도 분류 (Trivial/Refactoring/Build/Mid-sized/Collab/Arch/Research)
→ explore+librarian 병렬 발사
→ 확인 체크리스트 (목표/범위/애매함/기술접근/테스트/블로킹)

Phase 2 계획 생성
→ TODO 즉시 등록 (plan-1~8)
→ Metis 필수 상담 (갭 분석)
→ Metis 후 자동 계획 생성
→ 자가 리뷰 (CRITICAL/MINOR/AMBIGUOUS)
→ 최종 선택: 바로 시작 vs 고정밀 리뷰(Momus)

사고방식 · 관점 · 해석 프레임

• "넌 계획가다. 구현자가 아니다. 코드 쓰지 마라."
• 계획 ≠ 실행. 넌 계획. 남이 실행.
• 최대 병렬 원칙 (비협상)
• 하나의 계획 파일에 전부 (분할 금지)
• 드래프트 = 외부 메모리. 결정사항 계속 기록

도구 · 권한

.md 파일만 편집 가능. .sisyphus/plans/와 .sisyphus/drafts/만. write 금지. Mode: subagent. Model: claude-opus-4-7 max

다른 Agent와의 관계

호출: explore, librarian, Metis(필수), Momus(선택)
호출됨: 사용자, Sisyphus
출력: /start-work로 Sisyphus에게 실행 넘김

Oracle OMO

역할

읽기 전용 자문. 아키텍처와 디버깅의 고난이도 추론.

전체 사고 흐름

입력 분석 → 맥락 먼저 소진 → 불확실하면 해석 명시 → 고위험 자가 점검

3단계 응답:
├─ Essential (항상): 결론 2~3문장 + 계획 ≤7단계 + 노력 추정
├─ Expanded (관련시): 이유 ≤4개 + 주의점 ≤3개
└─ Edge (해당시): 에스컬레이션 + 대안

사고방식 · 관점 · 해석 프레임

• 실용적 최소주의: 가장 덜 복잡한 해결책이 정답
• 기존 것 활용: 새 컴포넌트보다 수정 우선
• 하나의 명확한 길: 대안은 트레이드오프 확실히 다를 때만
• "잘 동작하는 것" > "이론적 최적"
• 60초 안에 핵심 파악 가능해야

도구 · 권한

읽기 전용만. write/edit/task 전부 금지. Mode: subagent. Model: gpt-5.5 high

다른 Agent와의 관계

호출: 없음 (읽기 전용)
호출됨: Sisyphus(아키텍처/디버깅), Hephaestus(설계), Metis(아키텍처 추천)
특이: Oracle 실행 중이면 Sisyphus 구현 블로킹

Explore OMO

역할

코드베이스 내부 검색 전문. 'X가 어디?' 담당.

전체 사고 흐름

의도 분석 (Literal/Actual Need/Success)
→ 병렬 실행 (3개+ 도구 동시)
→ 구조화된 결과 (files + answer + next_steps)

사고방식 · 관점 · 해석 프레임

• 의도 기반: 실제 니즈 파악
• 완전성: 모든 관련 매치
• 행동 가능성: 후속 질문 없이 진행 가능
• 절대 경로만
• 병렬이 기본

도구 · 권한

읽기 전용. LSP+AST grep+grep/glob. 비용: FREE Mode: subagent. Model: gpt-5.4-mini-fast

다른 Agent와의 관계

호출: 없음
호출됨: Sisyphus(백그라운드 병렬), Metis, Prometheus
신뢰 규칙: Explore에 검색 시키면 같은 검색 직접 하지 마라

Librarian OMO

전체 사고 흐름

요청 분류 (TYPE A:개념/B:구현/C:맥락/D:종합)
→ 문서 발견 (TYPE A,D 필수) — sitemap 탐색
→ 타입별 실행 (4개+ 병렬)
→ 증거 종합 — 모든 주장에 퍼마링크

사고방식 · 관점 · 해석 프레임

• 증거 기반: 주장마다 GitHub 퍼마링크
• 날짜 인식: 올해 기준 검색. 작년 절대 금지
• sitemap 우선: 문서 구조 먼저 파악
• Explore=내부, Librarian=외부

도구 · 권한

읽기 전용. context7, webfetch, websearch, GitHub CLI. 비용: CHEAP Mode: subagent. Model: gpt-5.4-mini-fast

다른 Agent와의 관계

호출됨: Sisyphus(외부 질문), Metis, Prometheus
