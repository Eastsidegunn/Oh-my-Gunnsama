OmG v1 Core DB
DBML 그대로 구현한 현황

수정된 구현 기준 문서다. 이전 HTML의 trace-only VictoriaTraces 설명은 폐기. 현재 구현은 .omx/plans/omg-v1-core-db.dbml의 중심 객체를 repo-local .omg/omg.db SQLite schema로 생성하고, orchestrator/verify가 DB-backed Event Ledger와 관련 table에 기록한다.

Default DB: .omg/omg.dbDecision: DEC-010JSONL 없음DBML-shaped tables 구현

1. 지금 구현된 것

DB schema

  • 30개 DBML table 생성
  • enum은 SQLite CHECK로 반영
  • 핵심 FK/unique/index 반영

Runtime wiring

  • cmd/omg daemon start가 DB open/migrate
  • orchestrator에 storage.Store 주입
  • 종료 시 DB close

Write path

  • Run/Event Ledger 기록
  • ToolCall specialized row 기록

5. Deep Module 단위

internal/storage

  • OpenStore(ctx, projectRoot, Options): .omg/omg.db 생성
  • Migrate(): DBML-shaped schema 생성
  • RecordEvent(): Event Ledger + specialized table write
  • Count/HasTable: 검증용 query

cmd/omg

  • daemon start 시 DB를 항상 open/migrate
  • config storage.database.path로 DB path override 가능
  • orchestrator dependency로 DB store 주입

internal/orchestrator

  • submit → run_created
  • tool-before → tool_call_started/blocked
  • tool-after → tool_call_completed/failed
  • error/stop → run_status_changed

internal/verify

  • verification result → verification_passed/failed
  • check evidence → evidence_created
  • DB sink 없으면 기존 behavior 유지
