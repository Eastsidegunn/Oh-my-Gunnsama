OmG v1 Storage
VictoriaTraces 구현 맵

현재 구현된 범위를 C4, DB/Trace 모델, 모듈 단위로 정리한 문서다. 핵심은 JSONL source-of-truth가 아니라 VictoriaTraces-backed event/trace DB 방향이며, DBML 전체 migration이 아니라 v1 kernel만 잘라서 붙인 상태다.

Status: implemented kernel + opt-in daemon wiring Decision: DEC-009 accepted Verified: go test ./... + go vet ./... Date: 2026-05-16 Asia/Seoul

1. Executive Summary

구현됨

  • internal/storage: v1 semantic event kernel
  • VictoriaTracesSink: OTLP/HTTP span export
  • orchestrator hooks: run/tool/state event emit
  • verify hooks: verification/evidence event emit
  • daemon opt-in wiring: config enabled일 때 sink 주입

의도적으로 안 함

  • DBML 전체 migration 없음
  • JSONL source-of-truth 없음
  • PostgreSQL 전제 없음
  • VictoriaTraces process supervision 없음
  • query/read side 없음

현재 저장 의미

OmG 내부 이벤트는 storage.Event로 정규화되고, enabled 상태의 daemon에서는 VictoriaTracesSink가 이를 OpenTelemetry span으로 변환해 VictoriaTraces OTLP endpoint로 보낸다.

6. Deep Module View

cmd/omg

  • runDaemonStart()가 config와 registry를 검증한다.
  • buildStorageSink()가 storage.victoriatraces.enabled일 때만 sink 생성.
  • 생성된 sink를 orchestrator.Dependencies.StorageSink로 주입.
  • daemon 종료 시 Shutdown().

internal/config

  • Config.Storage.VictoriaTraces 추가.
  • 기존 built-in → user → project merge 흐름을 그대로 사용.
  • 지원 key: enabled, endpoint, service_name, storage_subdir.

internal/orchestrator

  • StorageSink storage.Sink dependency 추가.
  • sink가 nil이면 no-op.
  • protocol.Request payload에서 run_id, work_id, tool_call_id 등을 추출.
  • guard/tool/state transition 이벤트를 semantic event로 기록.

internal/verify

  • Options.StorageSink 추가.
  • verification summary를 verification_passed/failed로 기록.
  • 각 check result를 evidence_created로 기록.
  • sink nil이면 기존 verification behavior 유지.

internal/storage

  • Event: DBML vocabulary cut.
  • Recorder: timestamp/default validation + sink dispatch.
  • Sink: storage backend boundary.
  • VictoriaTracesConfig: endpoint/service/data path defaults.

internal/storage VictoriaTracesSink

  • OTLP/HTTP exporter 생성.
  • 각 semantic event를 omg.<event_type> span으로 변환.
  • omg.* attributes로 run/work/tool/evidence IDs 저장.
  • failed/blocked status는 OpenTelemetry error status로 표시.
